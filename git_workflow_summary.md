# Git Workflow для проекта TUUS

## Использованные команды Git
- git init - инициализация репозитория
- git add - добавление файлов в staging area
- git commit - сохраняет текущие изменения
- git checkout - переключение между ветками
- git checkout -b - создание новой ветки и переключение на нее
- git branch - просмотр веток
- git log - просмотр историй коммитов
- git push - отпарвляет коммиты в удаленный репозиторий

## Созданные ветки
- main - основная ветка проекта
- feature/add-discount-system - ветка для системы скидок
- feature/add-inventory-management - ветка для управления складом
- feature/add-shipping - ветка для расчета доставки

## Разрешенные конфликты
- Конфликт в src/models/product.py при слиянии feature/add-discount-system с main
- Причина конфликта: изменения были внесены  в один и тот же участок кода
в разных ветках
- Конфликт был разрешен вручную, путем объединения двух методов:
calculate_shipping() и get_category()
- После разрешения конфликта изменения были добавлен и выполнен git commit

## Стратегия работы с ветками
- Для каждой новой функции создается отдельная feature-ветка
- Ветки создаются от основной ветки main
- Вся разработка ведется в feature-ветке
- После выполнения работы изменения фиксируются с помощью git add и git commit
- Перед слиянием ветка обновляется из main через git merge main
- При возникновении конфликтов они решаются вручную
